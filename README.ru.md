# Первичная настройка CentOS (только домашняя лаборатория)

**English:** [README.md](README.md)

Здесь содержится Ansible playbook для первичной настройки операционной системы CentOS.

⚠️ **Отказ от ответственности:**
Эта настройка предназначена только для тестовых и среды разработки.
Для продакшена не рекомендуется. Перед выкладкой в продакшен просмотрите, адаптируйте и ужесточите конфигурации.

## Перед запуском

1. **`inventory.ini`**  
   Укажите `ansible_host` — IP или DNS целевого сервера. Группа `[servers]` и имя хоста (`server`) в примере — заглушки, при необходимости переименуйте. В `ansible_ssh_private_key_file` должен быть путь к **приватному** ключу для **первичного** входа под **root** (если для нового пользователя используется та же пара ключей, публичная часть должна попасть в `authorized_keys`).

2. **`centos_init_setup/defaults/main.yml`**  
   Замените `new_user` (по умолчанию `username`) на нужное имя пользователя. Доступ по ключу задаётся **либо** строкой в `ssh_public_key`, **либо** путём в `ssh_public_key_file`, который Ansible сможет прочитать на узле, с которого запускается playbook. При запуске через Docker ниже `~/.ssh` монтируется в `/root/.ssh` контейнера, поэтому путь вида `/root/.ssh/id_rsa.pub` обычно надёжнее, чем `~/.ssh/...`.

3. **Проверка**  
   Убедитесь, что до запуска playbook выполняется `ssh root@<хост>` с этим ключом. Роль отключает вход root и аутентификацию по паролю по SSH; после успешного прогона подключаться нужно как `new_user`.

4. **По желанию**  
   Просмотрите `centos_init_setup/tasks/ssh.yml` и `centos_init_setup/tasks/system.yml`, если нужны другие параметры усиления SSH или автоматических обновлений.

## Использование

Если `inventory.ini` и переменные роли настроены и есть SSH-доступ к root по ключу, запускайте playbook так:

```bash
docker run -it --rm \
  -v ~/.ssh:/root/.ssh \
  -v $(pwd):/apps \
  -w /apps \
  alpine/ansible:2.17.0 \
  ansible-playbook -i inventory.ini setup.yaml
```

Playbook использует роль `centos_init_setup`. Переменные `new_user`, `ssh_public_key` и `ssh_public_key_file` можно задать в `centos_init_setup/defaults/main.yml`, через `-e "new_user=…"`, либо через `host_vars` / `group_vars`.

## Примечания

Playbook выполнит:

*   Обновление пакетов системы
*   Создание пользователя с sudo и доступом по SSH-ключу
*   Базовое усиление SSH (отключение входа root, аутентификации по паролю)
*   Настройку автоматических обновлений (по умолчанию выключено — закомментировано; включите, раскомментировав соответствующие строки в `centos_init_setup/tasks/system.yml`)
*   Настройку PS1 и алиасов

Перед использованием всегда тестируйте вне продакшена.
